## 🕊️ The Mission
**POS ni Estela** is a bespoke, high-density Point of Sale and Inventory Intelligence system built specifically for my mother’s retail operations. It combines "Instant-On" mobile efficiency with a clean, expansive "Executive Terminal" aesthetic for desktop, ensuring that store data is not just recorded, but understood.

---

## 🧠 Intelligence Suite

### 🤖 AnimaBot: The Guardian
Our built-in **Shop Health Sentinel**. AnimaBot doesn't just display data; it interprets it. 
- **Contextual Feedback**: Randomizes messages based on daily profit, stock levels, and transaction velocity.
- **Emotional Logic**: Transitions between *Neutral*, *Healthy*, *Hard Work*, and *Risk* states to provide a human-centric read on the store's pulse.
- **Historical Time-Travel**: Syncs with the horizontal calendar to give sentiment feedback on specific past dates.

### 📈 Metrics Radar
- **Dual-Radial Performance**: High-fidelity SVG gauges tracking **Profit Goals** vs. **Yesterday's Benchmark**.
- **Monthly Yield Flow**: Advanced Chart.js visualizations with a 2,200-peso ceiling to monitor long-term growth trajectories.
- **Dynamic Goals**: Real-time tracking against a ₱500 daily profit mission.

## 🚀 Technical Stack

- **Frontend**: Next.js 15 (App Router) + React 19
- **Database/Auth**: Supabase (PostgreSQL)
- **Visuals**: Chart.js for canvas-level reporting + Lucide icons
- **Animations**: Lottie + Framer Motion for premium micro-interactions
- **Styling**: Tailored Tailwind CSS with a unified "Technical Premium" theme

## 🛠️ Setup & Development

### 1. Clone & Install
```bash
git clone [repository-url]
cd pos
npm install
```

### 2. Environment Configuration
Create a `.env.local` with your Supabase credentials:
```env
NEXT_PUBLIC_SUPABASE_URL=your_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key
```

### 3. Ignition
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) to access the command center.
